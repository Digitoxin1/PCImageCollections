Footnotes
  1. Verified Images:  DTC.exe reports that all tracks are unmodified or verification has been confirmed by two or more matching dumps
  2. Unverified Images:  These images are either unmodified or have been restored to an unmodified state. Restoration may include one or more of the following:
    1. Boot sector OEM name has been restored from modifications made by Windows 95/98
    2. Creation and Last Accessed dates added by Windows have been removed
    3. Files added to the disk by installation programs, the application itself, or a user have been removed where possible
    4. Data from a different release of the same revision of the application may have been used to help restore the image
    5. Miscellaneous data in the unused portions of the disk has been removed if it was believed it didn't belong there in the first place
    Note: a. and b. are the most common situations and make up the majority of any restoration that needs to be done. There is no guarantee that any restoration work has restored the disk completely back into an unmodified state. Additional dumps are always needed to verify the state of the image.
  3. Copy Protected Images:  .IMA/.IMG images cannot be used unless cracked and are provided for archival use only. .MFM and .86F images are included in the archive when available and are supported by the 86Box emulator.
